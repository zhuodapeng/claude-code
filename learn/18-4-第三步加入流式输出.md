# 18-4 第三步：加入流式输出

## 目标

让 LLM 的回答实时出现——每生成一个词，立刻打印到屏幕，而不是等整个回答生成完再一次性显示。

这就是我们在 Claude Code 里看到的"打字机"效果。

---

## 流式 vs 非流式：技术差异

**非流式**（step2 的做法）：

```
Claude Code                    API
    │── messages.create() ────►│
    │                          │ 等待... (可能几秒)
    │◄─── 完整响应 ─────────────│
    │ 打印全部文本
```

**流式**：

```
Claude Code                    API
    │── messages.stream() ────►│
    │◄─── text_delta "我" ──────│  立刻打印 "我"
    │◄─── text_delta "来" ──────│  立刻打印 "来"
    │◄─── text_delta "帮" ──────│  立刻打印 "帮"
    │◄─── text_delta "你" ──────│  ...
    │◄─── message_stop ─────────│  完成
```

用户体验从"等待→一次性显示"变成了"立即开始显示→逐渐出现"。

---

## 流式 API 的事件类型

Anthropic SDK 的流式 API 会发出一系列事件（第 5 章、第 9 章已经详细讲过）。关键事件：

| 事件 | 含义 |
|------|------|
| `message_start` | 新的 assistant 消息开始 |
| `content_block_start` | 内容块开始（text 或 tool_use）|
| `content_block_delta` | 内容增量（text_delta 或 input_json_delta）|
| `content_block_stop` | 内容块结束 |
| `message_stop` | 消息结束 |

对我们来说关键的是 `content_block_delta`：
- 类型 `text_delta`：LLM 的文字回答
- 类型 `input_json_delta`：工具调用参数（JSON 片段）

## 完整代码：step3.ts

```typescript
// step3.ts - 加入流式输出
import Anthropic from '@anthropic-ai/sdk'
import * as readline from 'readline'
import * as fs from 'fs/promises'
import { exec } from 'child_process'
import { promisify } from 'util'

const execAsync = promisify(exec)

const TOOLS: Anthropic.Tool[] = [
  {
    name: 'read_file',
    description: '读取文件内容',
    input_schema: {
      type: 'object',
      properties: { path: { type: 'string', description: '文件路径' } },
      required: ['path'],
    },
  },
  {
    name: 'bash',
    description: '执行 bash 命令',
    input_schema: {
      type: 'object',
      properties: { command: { type: 'string', description: '命令' } },
      required: ['command'],
    },
  },
]

async function executeTool(name: string, input: Record<string, unknown>): Promise<string> {
  if (name === 'read_file') {
    return fs.readFile(input.path as string, 'utf-8').catch(e => `Error: ${e}`)
  }
  if (name === 'bash') {
    return execAsync(input.command as string, { timeout: 30000 })
      .then(({ stdout, stderr }) => stdout + (stderr ? '\nSTDERR: ' + stderr : ''))
      .catch(e => `Error: ${e.message}`)
  }
  return `Unknown tool: ${name}`
}

const client = new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY })
const messages: Anthropic.MessageParam[] = []

async function ask(userInput: string): Promise<void> {
  messages.push({ role: 'user', content: userInput })

  process.stdout.write('\nAssistant: ')

  while (true) {
    // ======= 关键变化：用 stream() 替代 create() =======
    const stream = await client.messages.stream({
      model: 'claude-opus-4-6',
      max_tokens: 8096,
      tools: TOOLS,
      messages: messages,
    })

    // 收集本次流式响应的完整内容，用于后续处理
    const contentBlocks: Anthropic.ContentBlock[] = []

    // 追踪当前正在构建的工具调用（可能同时有多个）
    const toolCallAccumulators: Map<number, {
      id: string
      name: string
      inputJson: string
    }> = new Map()

    let stopReason: string | null = null

    // ======= 逐事件处理 =======
    for await (const event of stream) {
      if (event.type === 'message_start') {
        // 消息元数据，暂时不需要处理
      }

      else if (event.type === 'content_block_start') {
        const block = event.content_block
        if (block.type === 'text') {
          // text block 开始，等待 text_delta
        }
        else if (block.type === 'tool_use') {
          // 工具调用 block 开始，初始化 accumulator
          toolCallAccumulators.set(event.index, {
            id: block.id,
            name: block.name,
            inputJson: '',
          })
        }
      }

      else if (event.type === 'content_block_delta') {
        const delta = event.delta

        if (delta.type === 'text_delta') {
          // 立刻打印文字 token
          process.stdout.write(delta.text)
        }

        else if (delta.type === 'input_json_delta') {
          // 工具调用的 JSON 参数是流式的，需要累积
          const accumulator = toolCallAccumulators.get(event.index)
          if (accumulator) {
            accumulator.inputJson += delta.partial_json
          }
        }
      }

      else if (event.type === 'content_block_stop') {
        const accumulator = toolCallAccumulators.get(event.index)
        if (accumulator) {
          // 工具调用参数积累完毕，解析 JSON
          try {
            const input = JSON.parse(accumulator.inputJson)
            contentBlocks.push({
              type: 'tool_use',
              id: accumulator.id,
              name: accumulator.name,
              input,
            })
          } catch (e) {
            console.error(`\nFailed to parse tool input: ${accumulator.inputJson}`)
          }
          toolCallAccumulators.delete(event.index)
        }
      }

      else if (event.type === 'message_delta') {
        // 包含 stop_reason
        stopReason = event.delta.stop_reason ?? null
      }

      else if (event.type === 'message_stop') {
        // 流结束
      }
    }

    // 获取完整的最终消息（SDK 提供了便捷方法）
    const finalMessage = await stream.finalMessage()

    if (stopReason === 'end_turn' || !stopReason) {
      messages.push({ role: 'assistant', content: finalMessage.content })
      process.stdout.write('\n')
      return
    }

    if (stopReason === 'tool_use') {
      messages.push({ role: 'assistant', content: finalMessage.content })

      // 找出所有工具调用
      const toolUseBlocks = finalMessage.content.filter(
        (b): b is Anthropic.ToolUseBlock => b.type === 'tool_use'
      )

      const toolResults: Anthropic.ToolResultBlockParam[] = []

      for (const block of toolUseBlocks) {
        process.stdout.write(`\n[Tool: ${block.name}] ${JSON.stringify(block.input)}`)

        const result = await executeTool(
          block.name,
          block.input as Record<string, unknown>,
        )
        toolResults.push({
          type: 'tool_result',
          tool_use_id: block.id,
          content: result,
        })
      }

      messages.push({ role: 'user', content: toolResults })
      process.stdout.write('\nAssistant: ')
      // 继续循环，让 LLM 基于工具结果继续生成
    }
  }
}

async function main() {
  const rl = readline.createInterface({ input: process.stdin, output: process.stdout })
  console.log('LLM CLI with streaming ready.\n')

  for await (const line of rl) {
    if (!line.trim()) continue
    await ask(line)
    console.log()
  }
}

main().catch(console.error)
```

## 理解工具调用的 JSON 流式传输

工具调用参数是 JSON，但 JSON 也是流式传来的！

LLM 生成工具调用时，参数 JSON 不是一次性传来的，而是分片：

```
input_json_delta: '{"co'
input_json_delta: 'mman'
input_json_delta: 'd":"ls'
input_json_delta: ' -la"}'
```

所以我们用 `inputJson` 字符串累积这些片段，等 `content_block_stop` 时再 `JSON.parse()`。

**文件：[src/core/queryLoop.ts](../src/core/queryLoop.ts)**（概念位置）

Claude Code 做了完全相同的事：

```typescript
// 在 Claude Code 里处理 input_json_delta 的逻辑
if (event.type === 'content_block_delta' && event.delta.type === 'input_json_delta') {
  currentToolUseBlock.rawInputAccumulator += event.delta.partial_json
}
```

## `stream.finalMessage()` 便捷方法

Anthropic SDK 提供了 `stream.finalMessage()` 来获取完整的最终消息对象，而不需要自己手动拼接：

```typescript
const finalMessage = await stream.finalMessage()
// finalMessage.content 是完整的 ContentBlock 数组
// finalMessage.stop_reason 是停止原因
```

我们在流式处理事件的同时，也调用了 `finalMessage()`。这是 SDK 的便利设计：你可以既实时处理事件（用于 UI 更新），又在最后拿到完整的消息对象（用于加入消息历史）。

## 本章小结

流式输出的核心变化：
1. `messages.create()` → `messages.stream()`
2. `for await (const event of stream)` 逐事件处理
3. `text_delta` 立刻打印到终端
4. `input_json_delta` 积累成完整 JSON 后执行
