# 1-1 这个项目是什么，为什么存在

> **本节目标**：建立对 Claude Code 的整体认知——它是什么、解决什么问题、为什么用这种形式实现。

---

## 一个问题引入

你有没有想过，当你在终端里敲下 `claude "帮我重构这段代码"` 的时候，背后究竟发生了什么？

不是"调用了 API"这么简单——任何人都能写 10 行代码调用 Anthropic API 拿回一段文字。**Claude Code 和那 10 行代码之间的差距，就是这整个项目存在的意义。**

这个差距包含：
- Claude 如何读取你本地的文件？（不是靠你粘贴，是它主动读的）
- Claude 如何执行 bash 命令？（真实执行，结果反馈给它，再继续思考）
- Claude 如何在一轮对话里自主完成多步骤任务？（循环调用，直到任务完成）
- 所有操作如何在**终端里实时显示**，而不是等几秒蹦出一大段文字？
- 你如何控制它"能做什么"，防止它删错文件？

这些问题的答案，构成了 Claude Code 这个项目的全部。

---

## 这个项目是什么

**Claude Code 是 Anthropic 官方出品的 AI Agent CLI（命令行界面）。**

一句话定义：它是一个**运行在你终端里的 AI 代理**，能够自主地读写文件、执行命令、调用外部服务，来完成你交代给它的编程任务。

注意几个关键词：

| 关键词 | 含义 |
|--------|------|
| **CLI** | 运行在终端里，不是 Web 应用，不是 IDE 插件 |
| **AI Agent** | 不是问答机器人，而是能**主动采取行动**的系统 |
| **自主** | 它能自己规划步骤，不需要你手动"下一步、下一步" |
| **读写文件/执行命令** | 真实地操作你的本地环境，不是沙盒模拟 |

这个项目是 Anthropic 的生产级代码，你手上这份是本地化运行版本（去掉了一些需要内部基础设施的功能），但核心架构完全保留。

## 为什么不直接调 API？

这是理解这个项目存在意义的关键问题。

**朴素方案**：写个脚本，读用户输入，调 Anthropic API，打印结果。完事。

**为什么不行**：

大语言模型本身只是一个**文本转文本**的函数：给它一段文字，它输出一段文字。它不能：
- 访问你的文件系统
- 执行任何命令
- 知道当前目录是什么
- 记住上一轮说了什么（不主动管理的话）

想让 Claude 真正"帮你写代码"，你需要解决一系列工程问题：

```
问题一：Claude 怎么"看到"你的代码？
  → 你要把文件内容读出来，放进对话上下文里

问题二：Claude 说"运行一下 npm test 看看"怎么办？
  → 你要执行这个命令，把输出再喂给 Claude

问题三：一个任务需要多步，Claude 怎么知道什么时候停？
  → 你要实现一个循环：调用 API → 执行工具 → 把结果放回 → 再调用 API

问题四：这个循环怎么在终端实时展示？
  → 你要实现流式渲染，逐 token 显示

问题五：Claude 要删文件时用户怎么确认？
  → 你要实现权限系统，拦截危险操作

……
```

**这些问题的完整工程解法，就是 Claude Code。**

## 这个项目的技术规格

在深入之前，先建立一个整体印象：

| 维度 | 数值 |
|------|------|
| 主语言 | TypeScript |
| 运行时 | Bun（非 Node.js） |
| 代码量 | 约 15 万行 |
| 最大单文件 | `src/main.tsx`（4690 行） |
| 工具数量 | 40+ 个 |
| 核心文件 | `src/query.ts`（70KB，核心循环） |

`src/main.tsx` 有 4690 行。这不是设计失误——它是整个 CLI 的初始化入口，处理 20+ 种启动路径，每条路径都在这里分叉。第 6 章会专门拆解它。

## 这个项目解决了什么"不可能三角"

大多数 AI 工具在以下三点里只能选两个：

```
        强大
        /\
       /  \
      /    \
     /      \
   安全 ──── 易用
```

- **强大 + 易用**（不安全）：直接给 AI 无限权限，能做任何事，但你睡不着觉
- **强大 + 安全**（不易用）：每个操作都要你手动确认，等于没有 AI
- **安全 + 易用**（不强大）：只能读只读，不能真正执行任何操作

Claude Code 的解法是**分层权限 + 用户可控**：

- 默认"遇到危险操作必须问用户"（安全）
- 用户可以选择"这类操作以后都允许"（易用）
- 存在 YOLO 模式 (`--dangerously-skip-permissions`)（强大，但你知道风险）

这个权限系统的完整实现在第 11 章。
