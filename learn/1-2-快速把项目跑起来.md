# 1-2 快速把项目跑起来

> **本节目标**：在你的机器上成功启动 Claude Code，感受它的运行形态，为后续读代码建立"感性认知"。

---

## 先跑起来，再读代码

学习一个项目最常见的错误是：一头扎进代码，读了两个小时，但完全不知道自己读的东西"长什么样"。

正确的顺序是：**先跑起来，感受它的行为，再带着感性认知去读代码。**

## 环境准备

### 安装 Bun

这个项目用 **Bun** 作为运行时（不是 Node.js）。Bun 是一个更快的 JavaScript/TypeScript 运行时，自带包管理器。

```bash
# macOS / Linux
curl -fsSL https://bun.sh/install | bash

# Windows（PowerShell）
powershell -c "irm bun.sh/install.ps1 | iex"
```

安装后验证：
```bash
bun --version   # 应该输出类似 1.x.x
```

### 配置 Anthropic API Key

你需要一个 Anthropic API Key。在 [console.anthropic.com](https://console.anthropic.com) 注册并创建。

在项目根目录创建 `.env` 文件：

```
ANTHROPIC_API_KEY=sk-ant-xxxxxxxxxxxxxxxxxxxx
```

项目启动脚本 [bin/claude-haha](../bin/claude-haha#L16) 用 `--env-file=.env` 参数把这个文件注入运行时：

```bash
exec bun --env-file=.env ./src/entrypoints/cli.tsx "$@"
```

这意味着 `.env` 里的变量会自动成为环境变量，代码里通过 `process.env.ANTHROPIC_API_KEY` 读取。

### 安装依赖

```bash
cd /path/to/claude-code
bun install
```

## 启动

```bash
# 方式一：直接运行（推荐）
bun run start

# 方式二：通过 bin 脚本
./bin/claude-haha

# 方式三：直接启动入口
bun --env-file=.env ./src/entrypoints/cli.tsx
```

这三种方式等价，最终都走到同一个入口文件。

## 你会看到什么

启动成功后，终端显示的是一个完整的 **TUI（终端用户界面）**，用 React + Ink 渲染：

```
╭─────────────────────────────────────────────────────╮
│  Claude Code  v999.0.0-local                        │
│  Working directory: /your/project                   │
╰─────────────────────────────────────────────────────╯

> █                          ← 光标在这里等你输入
```

这不是普通的终端输出——它是 **React 组件树渲染到终端**的结果。你输入文字时，界面实时更新；Claude 在"打字"时，你能看到文字一个个出现。第 4 章会详细讲 Ink 如何做到这一点。

## 几个值得观察的行为

### 观察 1：Claude 主动读文件

输入：
```
你好，这个项目是做什么的？简单看看 package.json 和 README
```

观察 Claude 的行为：它会**主动调用工具**来读取文件，你会在界面上看到类似：

```
● Read(package.json)               ← 工具调用，用 ● 标记
● Read(README.md)

基于 package.json 和 README，这个项目是……
```

这个"读文件"的动作不是 Claude 凭空猜测的——它是通过**工具调用（Tool Use）** 真实执行的，第 7 章会详细讲工具系统。

### 观察 2：流式输出

Claude 的回答不是一次性蹦出来的——文字是**一个 token 一个 token 地出现**，就像有人在实时打字。这是**流式响应**的效果，第 5-2 节会讲它的技术原理。

### 观察 3：命令执行与权限确认

输入：
```
列出当前目录的文件
```

你会看到 Claude 想执行 `ls` 命令，但在执行前会询问：

```
● Bash(ls -la)

  Allow bash command: ls -la?
  ❯ Yes, allow once
    Yes, don't ask again
    No
```

这是**权限系统**在工作。Claude 不能"偷偷"执行命令——每个操作都要经过权限检查。第 11 章会深入讲这个系统。

### 观察 4：多步自主执行

输入一个需要多步完成的任务：
```
创建一个 hello.ts 文件，写一个 hello world 程序，然后运行它
```

观察 Claude 如何**自主规划步骤**：
1. 先创建文件（Write 工具）
2. 再运行它（Bash 工具）
3. 如果报错，自动调试

这种自主的多步执行就是 **Agent 行为**，驱动它的是 `queryLoop`，第 9 章核心内容。

## 项目的几种启动模式

[bin/claude-haha](../bin/claude-haha) 脚本非常简短，但揭示了两种启动路径：

**文件：[bin/claude-haha](../bin/claude-haha#L1-L17)**

```bash
#!/usr/bin/env bash
set -euo pipefail

ROOT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"
export CALLER_DIR="$(pwd -W 2>/dev/null || pwd)"   # 记录用户执行命令时所在的目录

cd "$ROOT_DIR"

# 路径一：强制恢复模式（简单的 readline REPL，无 TUI）
if [[ "${CLAUDE_CODE_FORCE_RECOVERY_CLI:-0}" == "1" ]]; then
  exec bun --env-file=.env ./src/localRecoveryCli.ts "$@"
fi

# 路径二：默认模式，完整 TUI
exec bun --env-file=.env ./src/entrypoints/cli.tsx "$@"
```

几个细节值得注意：

1. **`CALLER_DIR`**：记录用户执行命令时的目录（而不是项目根目录）。Claude 需要知道"你在哪个项目里调用我"，而不是"我自己在哪里"。

2. **`CLAUDE_CODE_FORCE_RECOVERY_CLI`**：当 Ink TUI 因为终端兼容性问题崩溃时，用户可以设置这个环境变量降级到简单 REPL。这是优雅降级的设计。

3. **`exec`**：用 exec 替换当前进程，而不是创建子进程。这样 Bun 进程直接继承了 shell 进程的 PID，Ctrl+C 信号能正确传递。

## 常见问题

**Q: 报错 `Cannot find module`**
A: 执行 `bun install` 安装依赖。

**Q: 报错 `ANTHROPIC_API_KEY is not set`**
A: 确认 `.env` 文件存在且 Key 正确。

**Q: 界面显示乱码**
A: 你的终端可能不支持 Unicode 或颜色。尝试用 iTerm2（Mac）或 Windows Terminal（Windows）。

**Q: 想直接用 `claude-haha` 命令**
A: 运行 `bun link`，会把 `claude-haha` 注册为全局命令。
