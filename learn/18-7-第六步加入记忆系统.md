# 18-7 第六步：加入记忆系统

## 目标

让 LLM 记住跨会话的重要信息：用户偏好、项目上下文、重要决策……

关闭再打开 CLI，上一次学到的东西不会消失。

---

## 记忆系统的本质

LLM 没有持久记忆——每次会话都从零开始。但我们可以在 system prompt 里注入之前保存的信息。

**核心机制**：

```
会话 1：
  用户："我叫 Alice，我在用 Python 写一个爬虫项目"
  LLM 执行 → 保存记忆："用户叫 Alice，Python 爬虫项目"

会话 2（新开 CLI）：
  System Prompt 注入："用户记忆：用户叫 Alice，Python 爬虫项目"
  用户："帮我优化一下代码"
  LLM："当然，Alice！你的爬虫项目..."
```

这和 Claude Code 的记忆系统（第 15 章）是一样的思路：
- 记忆存储在 `MEMORY.md`（或类似文件）
- 启动时读取，注入 system prompt
- 会话中可以更新

## 实现记忆系统

```typescript
// memory.ts

import * as fs from 'fs/promises'
import * as path from 'path'
import * as os from 'os'

// 记忆文件路径：~/.my-llm-cli/MEMORY.md
const MEMORY_DIR = path.join(os.homedir(), '.my-llm-cli')
const MEMORY_FILE = path.join(MEMORY_DIR, 'MEMORY.md')

// ======= 读取记忆 =======
export async function loadMemory(): Promise<string> {
  try {
    const content = await fs.readFile(MEMORY_FILE, 'utf-8')
    if (!content.trim()) return ''
    return content
  } catch (e) {
    // 文件不存在，返回空
    return ''
  }
}

// ======= 保存记忆 =======
export async function saveMemory(content: string): Promise<void> {
  await fs.mkdir(MEMORY_DIR, { recursive: true })

  // 原子写入：write to temp + rename
  const tempFile = MEMORY_FILE + '.tmp.' + Date.now()
  try {
    await fs.writeFile(tempFile, content, 'utf-8')
    await fs.rename(tempFile, MEMORY_FILE)
  } catch (e) {
    await fs.unlink(tempFile).catch(() => {})
    throw e
  }
}

// ======= 构建包含记忆的 System Prompt =======
export async function buildSystemPrompt(): Promise<string> {
  const memory = await loadMemory()

  let systemPrompt = `你是一个有用的 AI 助手，运行在终端里。
你可以使用工具来读取文件和执行命令。`

  if (memory) {
    systemPrompt += `

# 用户记忆

以下是从之前会话中保存的重要信息：

${memory}

---`
  }

  return systemPrompt
}
```

## 自动提取记忆的工具

关键问题：谁来决定"保存什么"？

**方案 A（手动）**：用户明确说"记住这个"，我们保存。简单，但依赖用户主动。

**方案 B（自动提取）**：每次对话结束后，让 LLM 分析对话，提取应该记住的内容。这是 Claude Code 的做法。

我们实现方案 B——加一个 `save_memory` 工具，让 LLM 自主决定何时保存：

```typescript
// 在 TOOLS 里加入记忆工具
{
  name: 'save_memory',
  description: `保存重要信息到持久记忆，跨会话保存。
使用场景：
- 用户透露了重要偏好（编程语言、代码风格）
- 用户描述了项目背景
- 做出了重要决策
- 用户明确要求记住某事
不要保存临时信息或会话内容。`,
  input_schema: {
    type: 'object',
    properties: {
      key: {
        type: 'string',
        description: '记忆的类别，如 "user_preference"、"project_context"',
      },
      content: {
        type: 'string',
        description: '要保存的内容，简洁明了',
      },
    },
    required: ['key', 'content'],
  },
}
```

执行逻辑：

```typescript
if (name === 'save_memory') {
  const { key, content } = input as { key: string; content: string }

  // 读取现有记忆
  const existing = await loadMemory()

  // 更新或追加
  let newMemory = existing
  const keyHeader = `## ${key}\n`

  if (existing.includes(keyHeader)) {
    // 更新现有条目（简单字符串替换）
    const regex = new RegExp(`## ${key}\\n[^#]*`, 'g')
    newMemory = existing.replace(regex, `${keyHeader}${content}\n\n`)
  } else {
    // 追加新条目
    newMemory = existing + `\n${keyHeader}${content}\n`
  }

  await saveMemory(newMemory.trim())
  return `已保存记忆：[${key}] ${content}`
}
```

## 记忆的注入时机

启动时，在第一次 API 调用前，把记忆注入 system prompt：

```typescript
// 在 App 组件的初始化里
const [systemPrompt, setSystemPrompt] = useState('')

useEffect(() => {
  buildSystemPrompt().then(setSystemPrompt)
}, [])

// 在 API 调用里
const stream = await client.messages.stream({
  model: 'claude-opus-4-6',
  max_tokens: 8096,
  system: systemPrompt,     // ← 注入记忆
  tools: TOOLS,
  messages: loopMessages,
})
```

## 记忆的截断

记忆文件可能越来越大。Claude Code（第 15-4 节）的做法是截断超过上限的部分。我们的简化实现：

```typescript
const MAX_MEMORY_TOKENS = 2000  // 约 2000 tokens 的预算

export async function buildSystemPrompt(): Promise<string> {
  let memory = await loadMemory()

  // 粗略估计：1 token ≈ 4 字符
  const MAX_MEMORY_CHARS = MAX_MEMORY_TOKENS * 4

  if (memory.length > MAX_MEMORY_CHARS) {
    // 简单截断：只保留最近的内容（末尾部分）
    // Claude Code 的实现更复杂：从末尾倒推，按段落截断
    memory = '...[older memories truncated]...\n\n' +
             memory.slice(-MAX_MEMORY_CHARS)
  }

  // ...
}
```

## 本章小结

记忆系统的核心设计：
1. **文件存储**：记忆持久化到磁盘（`MEMORY.md`）
2. **注入时机**：启动时读取，通过 system prompt 注入
3. **原子写入**：temp + rename，防止损坏
4. **LLM 自主决定**：`save_memory` 工具让 LLM 自主判断何时保存
