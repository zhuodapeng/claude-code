# 2-2 函数作为一等公民 + 模块系统

> **本节目标**：理解 TypeScript/JavaScript 的函数特性——函数可以作为参数、返回值、存储在变量里。这是 Claude Code 架构的基础模式之一。

---

## 什么是"函数作为一等公民"？

在 Java 里，函数（方法）必须附属于某个类。你不能把一个"裸函数"赋值给变量，或者把函数当成参数传给另一个函数（Java 8 的 Lambda 稍微打破了这个限制，但仍然受限）。

在 TypeScript/JavaScript 里，**函数就是普通的值**，和数字、字符串没什么本质区别。

```typescript
// 函数存储在变量里
const greet = (name: string) => `Hello, ${name}`

// 函数作为参数传递
function runTwice(fn: (x: string) => string, value: string) {
  console.log(fn(value))
  console.log(fn(value))
}
runTwice(greet, "world")

// 函数作为返回值
function makeMultiplier(factor: number) {
  return (x: number) => x * factor   // 返回一个新函数
}
const double = makeMultiplier(2)
double(5)  // 10
```

## 在 Claude Code 里，这个特性无处不在

### 场景 1：工具的 `call` 字段是一个函数

在 [src/Tool.ts](../src/Tool.ts#L379-L385) 里，`Tool` 类型的 `call` 就是一个函数：

**文件：[src/Tool.ts](../src/Tool.ts#L362-L385)**

```typescript
export type Tool<Input extends AnyObject = AnyObject, Output = unknown> = {
  name: string
  // call 是一个函数类型：接受参数，返回 Promise
  call(
    args: z.infer<Input>,
    context: ToolUseContext,
    canUseTool: CanUseToolFn,
    parentMessage: AssistantMessage,
    onProgress?: ToolCallProgress,
  ): Promise<ToolResult<Output>>
  // ...
}
```

每个工具都提供一个 `call` 函数，系统需要执行工具时就调用 `tool.call(args, context, ...)`。

### 场景 2：`canUseTool` 是一个函数参数

```typescript
type CanUseToolFn = (toolName: string, input: unknown) => Promise<PermissionResult>
```

这个函数被当作参数传给 `tool.call()`。工具执行时，如果要调用另一个工具，可以通过这个函数检查权限。

**本质**：你传给工具一个"检查权限的能力"，而不是让工具直接访问权限系统。这叫**依赖注入**——通过函数参数注入能力，而不是硬编码依赖。

## 箭头函数：最常见的写法

```typescript
// 完整写法
const add = function(a: number, b: number): number {
  return a + b
}

// 箭头函数（更简洁）
const add = (a: number, b: number): number => {
  return a + b
}

// 单表达式可以省略 return 和大括号
const add = (a: number, b: number) => a + b

// 无参数
const greet = () => "Hello"

// 返回对象（需要加括号，避免歧义）
const makeUser = (name: string) => ({ name, createdAt: Date.now() })
```

在项目代码里，你会大量看到箭头函数，特别是回调和短小的工具函数。

## 闭包：函数"记住"外部变量

这是 Java 开发者最容易忽视的特性：

```typescript
function makeCounter() {
  let count = 0    // 这个变量被"关"在了返回的函数里

  return {
    increment: () => { count++ },
    get: () => count,
  }
}

const counter = makeCounter()
counter.increment()
counter.increment()
counter.get()   // 2
```

`count` 不是全局变量，但 `increment` 和 `get` 函数都能访问它——因为它们共享了同一个**闭包作用域**。

**在 Claude Code 里的实际用法**：

工具的 `buildTool()` 工厂函数就是一个闭包模式——外部环境（配置、权限规则）在工厂函数里被闭合，返回的工具函数自动"记住"这些配置。第 7-3 节会详细讲。

## 高阶函数：接受/返回函数的函数

最常见的高阶函数：

```typescript
const numbers = [1, 2, 3, 4, 5]

// map：对每个元素应用函数，返回新数组
numbers.map(n => n * 2)          // [2, 4, 6, 8, 10]

// filter：保留满足条件的元素
numbers.filter(n => n % 2 === 0) // [2, 4]

// reduce：累积计算
numbers.reduce((sum, n) => sum + n, 0)  // 15

// find：找到第一个满足条件的元素
numbers.find(n => n > 3)         // 4
```

在 [src/tools.ts](../src/tools.ts) 的工具注册表里，大量使用 `.filter()`、`.map()`、`.find()` 来组装工具列表。

## 模块系统：`import` 和 `export`

TypeScript 使用 ES 模块（ESM）标准，和 Java 的 `import` 不同：

```typescript
// 具名导出（一个文件可以有多个）
export const TIMEOUT = 30000
export function helper() { ... }
export type Config = { ... }

// 默认导出（一个文件只能有一个）
export default class MyClass { ... }

// 导入具名
import { TIMEOUT, helper } from './config.js'

// 导入默认
import MyClass from './MyClass.js'

// 重命名导入（避免命名冲突）
import { helper as utilHelper } from './utils.js'

// 导入所有（不推荐，但有时用于命名空间）
import * as utils from './utils.js'
```

**注意：TypeScript 项目里 `import` 路径写 `.js` 扩展名**

这很反直觉——源码是 `.ts` 文件，但 import 路径写 `.js`。这是因为 TypeScript 在编译时会把 `.ts` 变成 `.js`，所以写的 import 路径要匹配编译产物。Bun 能直接处理这个，所以你会在代码里看到：

```typescript
import { buildTool } from '../../Tool.js'   // 源文件是 Tool.ts，但 import 写 .js
```

## 展开运算符：合并对象/数组

```typescript
// 合并对象（后面的属性覆盖前面的）
const merged = { ...defaultConfig, ...userConfig }

// 数组追加
const newMessages = [...existingMessages, newMessage]

// 函数参数展开
function sum(...numbers: number[]) {
  return numbers.reduce((a, b) => a + b, 0)
}
```

在消息历史管理里，`[...mutableMessages, newMsg]` 这种模式随处可见。

## 本节小结

- 函数是一等公民：可以存变量、当参数、当返回值
- 箭头函数是最常见的写法，单表达式可省略 `return`
- 闭包让函数"记住"定义时的环境，用于工厂模式
- 高阶函数（`map`、`filter`、`reduce`）替代大量 for 循环
- ESM 模块用 `import`/`export`，TypeScript 里 import 路径写 `.js`
