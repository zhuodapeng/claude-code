# 1-3 目录结构全景扫描

> **本节目标**：建立项目"地图感"——知道每个目录负责什么，避免在后续阅读中迷路。

---

## 为什么要先看目录结构？

代码库就像一座陌生的城市。在你去逛街之前，先看一眼地图，能让你知道：
- 商业区在哪里（核心逻辑）
- 住宅区在哪里（工具/基础设施）
- 什么地方先别去（暂时不重要的外围系统）

这一节，我们就画这张地图。

---

## 顶层结构

```
claude-code/
├── bin/                ← 可执行脚本（入口点）
├── src/                ← 所有源码（主战场）
├── learn/              ← 你正在读的课程
├── docs/               ← VitePress 文档
├── package.json        ← 项目依赖声明
├── .env                ← API Key（你自己创建）
└── bun.lock            ← 依赖锁文件
```

99% 的时间你都在 `src/` 里活动。

## src/ 内部结构

这是关键。下面按"重要程度"分层介绍：

### 第一层：必须掌握的核心文件

```
src/
├── main.tsx            ★★★  CLI 初始化总入口（4690 行）
├── query.ts            ★★★  核心循环：queryLoop（70KB）
├── QueryEngine.ts      ★★★  会话管理层（47KB）
├── Tool.ts             ★★★  工具基础接口定义
├── tools.ts            ★★★  工具注册表（assembleToolPool）
└── entrypoints/
    └── cli.tsx         ★★★  Bun 启动入口（快速路径优化）
```

这几个文件是整个系统的骨架，后续课程会逐个深入拆解。

### 第三层：外围支持系统

```
src/

├── remote/             ← 远程会话（WebSocket）
├── commands/           ← 斜杠命令（/help、/clear 等）
├── migrations/         ← 配置迁移
├── bootstrap/          ← 启动引导流程
├── keybindings/        ← 键盘快捷键

├── plugins/            ← 插件系统
└── bridge/             ← 远程控制桥接
```

## 这个项目的"心脏"在哪里

如果要指出这个项目最核心的文件，我会毫不犹豫地说：

```
src/query.ts     ← 心脏
```

所有的复杂性最终都汇聚在这里：
- 调用 Anthropic API
- 解析流式响应
- 触发工具执行
- 把工具结果喂回给 Claude
- 循环直到任务完成

理解了 `query.ts`，就理解了整个系统的运作方式。

---

## 各层级的职责关系

```
用户输入
    │
    ▼
src/main.tsx              ← 初始化，解析命令行参数
    │
    ▼
src/QueryEngine.ts        ← 会话管理：维护消息历史，处理多轮对话
    │
    ▼
src/query.ts              ← 核心循环：调用 API，执行工具，循环
    │                            ↕
    │              src/services/api/claude.ts  ← API 调用
    │                            ↕
    │              src/services/tools/         ← 工具执行编排
    │                            ↕
    │              src/tools/*/                ← 具体工具实现
    ▼
src/components/           ← 结果渲染为 TUI 界面（React + Ink）
```

## 本节小结

- 项目源码全在 `src/`，核心是 `main.tsx`、`query.ts`、`QueryEngine.ts`、`Tool.ts`
- `src/tools/` 是所有"能力"的载体（40+ 个工具）
- `src/services/` 是与外部通信的封装层（API、工具编排）
- 项目的"心脏"是 `src/query.ts` 中的 `queryLoop`
- 后续课程会逐层深入，从这张地图的不同区域展开
